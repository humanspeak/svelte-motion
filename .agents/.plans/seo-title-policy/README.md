## Findings considered and rejected

- **Apply only the four Ahrefs rewrites** — rejected because it would leave the
  68-character `/svelte-animations` title, preserve a 59-to-1 example suffix
  inconsistency, and add no guard against recurrence.
- **Keep `| Examples | Svelte Motion` and aggressively truncate the subjects**
  — rejected because the URL, breadcrumbs, and page content already establish
  the collection; spending 26 characters on repeated taxonomy weakens the
  distinctive, query-relevant part of each title.
- **Let only the three long examples omit `Examples`** — rejected because this
  is the exact inconsistency the maintainer dislikes and it creates a title
  policy nobody can predict from the route type.
- **Add a shared production title formatter** — rejected for this repository:
  docs-kit's `exampleMirrorsPlugin` statically parses literal `seo.title`
  assignments. Helper calls would break generated example mirrors unless the
  upstream parser were redesigned, which is disproportionate to this task.
- **Model Google truncation by pixel width** — rejected as a merge gate because
  display width varies by device and presentation. Use the 60-character rule
  to satisfy Ahrefs, then evaluate real title rewrites and click-through data
  in Search Console.
- **Rewrite H1, breadcrumbs, descriptions, and social-card titles together** —
  rejected because those fields are already concise and topically aligned.
  Mechanical identity across SEO surfaces is not a quality goal.
